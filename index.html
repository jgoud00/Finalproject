⚙ Client-Server Model — How Websites Work

Your browser is the client — it requests and displays pages. In a real portal, the browser sends your data to a server which queries a database and returns results. In this demo everything runs locally — scholarship data lives in data.js and the eligibility logic runs in app.js inside your browser. In real applications, this data would be fetched from a remote API over the internet.
